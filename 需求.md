ACF 管控系统 - 完整需求与用户故事

一、项目核心目标

打造一套全流程可视化的 ACF 胶料管控系统，实现从来料入库、存储、发料、退库到报废的全生命周期追踪，精准管控库存、有效期、使用次数与在库时长，通过预警与先进先出机制保障生产物料安全，同时用大屏看板实时呈现业务状态。
二、核心功能需求清单

1. 基础数据管理

料号与编码规则管理：自定义 ACF 料号、描述、单位、来料条码解析规则，支持不同厂商 / 型号的编码映射。
标签模板管理：固定标签尺寸（5cm×10cm），可配置字段（LOT 号、型号 / 规格、保存期限、使用记录栏位等），与打印驱动对接。
LOT 号规则管理：自定义 LOT 号生成规则（支持二维码 + 文字描述），LOT 号与 ACF 批次唯一绑定。
预警规则管理：自定义过期预警天数、库存低水位预警、使用次数超限预警（回温次数≤3 次）。
2. 来料与标签生成

来料扫码识别：扫描来料条码，自动匹配料号与对应 ACF 型号 / 规格，读取预设保存期限（如 6 个月）。
自动生成 LOT 号：按规则生成唯一 LOT 号（二维码 + 文本），填充至标签模板的「LOT」栏位，自动填入「型号 / 规格」「保存期限」。
标签打印：调用标签打印机，按 5cm×10cm 模板打印 ACF 管控卡（含使用记录栏位与备注）。
3. 库存与出入库管理

入库管理：扫描 LOT 号，绑定储位，记录入库时间（东七区曼谷时间），生成库存记录（包含数量、入账时间、有效期、在库天数）。
发料管理：扫描 LOT 号，执行发料动作，记录发料时间、操作人员，扣减库存，更新使用次数。
退库管理：扫描 LOT 号 + 储位，填写退库数量，记录退库时间，恢复库存，计算发料与退库的间隔时长。
全生命周期追踪：记录每卷 ACF 从入库到完全领用 / 报废的所有操作日志，包含入库、发料、退库、报废时间点与操作人员。
先进先出（FIFO）控制：发料时优先推荐入库时间最早、未过期的 LOT 号，禁止发放已过期或预警物料。
4. 库存监控与预警

实时库存查询：按料号、LOT 号、储位查询库存数量、在库天数、剩余有效期、使用次数。
多维度预警：
过期预警：距过期≤自定义天数时触发提醒。
使用次数预警：回温次数接近 3 次时提醒。
库存预警：库存低于预设安全水位时提醒。
报表统计：
库存台账报表（料号、LOT、数量、入库时间、有效期、在库天数、使用次数）。
出入库明细报表（发料 / 退库时间、间隔时长、操作人员）。
过期 / 预警物料清单。
所有报表数据均从数据库读取，前端仅做展示。
5. 可视化看板

大屏动态看板：滚动展示核心数据（总库存、待预警物料、过期物料、今日出入库数量、FIFO 执行情况）。
业务状态看板：分模块展示各型号 ACF 库存占比、有效期分布、使用次数统计。
6. 系统基础要求

时区统一：所有时间字段使用东七区（曼谷）时间。
数据存储：所有业务数据存储于数据库，前端不做复杂运算，仅做查询与展示。
操作日志：记录所有关键操作（入库、发料、退库、标签打印等），便于追溯。
三、用户故事（按业务流程）

1. 基础数据配置阶段

故事 1：作为系统管理员，我要配置 ACF 料号与条码解析规则，以便来料扫码时能自动识别对应型号。
故事 2：作为系统管理员，我要自定义 LOT 号生成规则，以便每批 ACF 都有唯一可追溯的标识。
故事 3：作为系统管理员，我要配置标签模板（5cm×10cm），确保打印出的标签与提供的模板格式一致。
故事 4：作为系统管理员，我要设置预警天数（如提前 7 天预警过期）与库存安全水位，以便及时发现风险物料。
2. 来料与标签打印阶段

故事 5：作为仓库操作员，我要扫描来料条码，系统自动匹配 ACF 型号 / 规格与保存期限，以便快速完成来料信息录入。
故事 6：作为仓库操作员，我要在系统中确认来料信息，系统自动生成唯一 LOT 号（二维码 + 文本），以便后续追溯。
故事 7：作为仓库操作员，我要点击打印按钮，系统调用标签打印机，输出符合模板的 ACF 管控卡，以便贴在物料上。
3. 入库阶段

故事 8：作为仓库操作员，我要扫描 LOT 号，选择储位并填写入库数量，系统记录入库时间（曼谷时间），以便建立库存台账。
故事 9：作为仓库操作员，我要查看入库成功提示，系统更新该 LOT 号的在库天数与库存数量，以便确认操作生效。
4. 发料阶段

故事 10：作为生产领料员，我要扫描目标 ACF 料号，系统推荐入库时间最早且未过期的 LOT 号（先进先出），以便合规领用物料。
故事 11：作为生产领料员，我要扫描推荐的 LOT 号，填写发料数量，系统扣减库存并记录发料时间，以便完成发料动作。
故事 12：作为生产领料员，我要查看发料成功提示，系统更新该 LOT 号的使用次数，以便监控回温次数是否超限。
5. 退库阶段

故事 13：作为生产领料员，我要扫描未用完的 ACF 的 LOT 号，选择储位并填写退库数量，系统记录退库时间，以便将物料放回库存。
故事 14：作为仓库操作员，我要查看退库记录，系统自动计算本次发料与退库的间隔时长，以便分析物料使用效率。
6. 库存监控与预警阶段

故事 15：作为仓库主管，我要在系统中查看实时库存报表，了解各型号 ACF 的库存数量、在库天数与有效期，以便掌握库存状态。
故事 16：作为仓库主管，我要接收系统推送的预警通知（过期 / 使用次数超限 / 库存不足），以便及时处理风险物料。
故事 17：作为生产主管，我要查看大屏看板，实时了解 ACF 库存、预警与出入库情况，以便保障生产连续性。
7. 报表与追溯阶段

故事 18：作为质量管理员，我要查询某 LOT 号的全生命周期记录，查看入库、发料、退库、报废的所有操作，以便进行质量追溯。
故事 19：作为财务 / 运营人员，我要导出库存台账与出入库明细报表，以便进行成本核算与业务分析。
四、用户行为路径（核心流程）

路径 1：来料→标签打印→入库

管理员提前配置料号、条码规则、标签模板与 LOT 规则。
仓库操作员扫描来料条码。
系统自动解析条码，匹配 ACF 型号 / 规格与保存期限。
操作员确认信息，系统生成唯一 LOT 号（二维码 + 文本）。
系统填充标签模板（LOT 号、型号 / 规格、保存期限等）。
操作员点击「打印标签」，系统调用打印机输出 5cm×10cm 管控卡。
操作员将标签贴于 ACF 物料上。
操作员扫描LOT 号，选择储位，填写入库数量。
系统记录入库时间（曼谷时间），生成库存记录。
系统更新在库天数、库存数量与有效期倒计时。
路径 2：发料→使用→退库

领料员扫描目标ACF 料号。
系统按先进先出原则，推荐入库时间最早、未过期的 LOT 号。
领料员扫描推荐的LOT 号，填写发料数量。
系统扣减库存，记录发料时间，更新使用次数。
领料员领用物料至产线使用。
若有剩余物料，领料员扫描LOT 号，选择储位，填写退库数量。
系统记录退库时间，恢复库存，计算发料与退库的间隔时长。
系统更新该 LOT 号的在库状态与使用次数。
路径 3：库存监控→预警处理→报表导出

主管登录系统，进入「库存监控」模块。
查看各型号 ACF 的库存数量、在库天数、有效期与使用次数。
系统自动高亮显示预警 / 过期物料，并推送通知。
主管处理预警（如优先发料预警物料、采购补货等）。
主管进入「报表中心」，选择需要的报表类型（库存台账 / 出入库明细 / 预警清单）。
系统从数据库读取数据，生成报表并支持导出。
路径 4：大屏看板查看

管理员配置大屏看板展示字段（总库存、预警数、今日出入库等）。
大屏自动滚动展示核心数据，实时更新库存与预警状态。
生产 / 仓库人员通过大屏直观了解业务运行情况。
五、补充优化建议

报废管理：增加报废流程，当 ACF 过期或使用次数超限后，扫描 LOT 号执行报废，标记为不可用，从库存中移除。
批次关联：支持将 LOT 号与生产工单绑定，便于追溯物料使用到哪批产品。
移动端适配：开发移动端页面，方便仓库人员在现场扫码操作（入库 / 发料 / 退库）。
权限控制：增加基础权限（如管理员 / 操作员 / 查看者），避免误操作。
回温记录自动化：在发料 / 退库时自动记录回温时间，满足备注中「回温时间≥30min」的要求。
六、技术实现关键点

时区处理：所有数据库时间字段存储为东七区时间，后端统一处理时区转换，避免时间混乱。
先进先出算法：发料时按入库时间升序排序，优先选择未过期、库存 > 0 的 LOT 号。
标签打印：通过打印指令（如 ZPL）驱动标签打印机，确保模板与尺寸精准匹配。
预警计算：后端定时任务（如每小时）计算所有 LOT 号的剩余有效期与使用次数，触发预警通知。
数据可视化：使用 ECharts 等库实现大屏看板的动态滚动与图表展示。
